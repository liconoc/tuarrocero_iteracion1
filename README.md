# Casos de uso

## Iteración 1

### 1. El usuario puede hacer login (email + contraseña)

El usuario debe poder iniciar sesión en el sistema utilizando su dirección de correo electrónico y contraseña.

### 2. El usuario puede listar los pedidos, así como ver, crear, editar, cancelar y borrar un pedido

El usuario debe poder ver una lista de todos los pedidos realizados. También debe poder ver, crear, editar, cancelar y borrar un pedido individual.

### 3. El usuario puede listar los repartidores, así como ver, crear, editar y borrar a un repartidor

El usuario debe poder ver una lista de todos los repartidores registrados en el sistema. También debe poder ver, crear, editar y borrar a un repartidor individual.

### 4. El usuario puede vincular un repartidor a un pedido

El usuario debe poder vincular un repartidor a un pedido específico. Esto se hace para que el repartidor pueda ver y realizar el pedido.

### 5. El usuario puede vincular una cocina a un pedido

El usuario debe poder vincular una cocina a un pedido específico. Esto se hace para que la cocina pueda preparar el pedido.

### 6. El usuario puede ver los pedidos servidos un día en concreto

El usuario debe poder ver una lista de todos los pedidos servidos en un día determinado.

### 7. El usuario puede cambiar manualmente el estado del pedido a alguno de los siguientes:

* **Recibido:** es el estado por defecto de un pedido cuando se crea.
* **En cocina:** es cuando el pedido se está realizando.
* **En reparto:** es cuando el pedido se está repartiendo.
* **Por devolver:** es cuando el pedido ha sido entregado al cliente y está pendiente de recoger.
* **En devolución:** es cuando el repartidor ha recogido la paella del cliente y queda pendiente de devolver a cocina.
* **Terminado:** es cuando la paella (vacía) ya ha vuelto a la cocina.
